# Contributing

## Principles

- preserve semantic parity with `kotlin-drmanhatan`
- keep the public API small and explicit
- avoid coupling the core package to vendor-specific SDKs
- prefer immutable records and small interfaces

## Suggested flow

1. Keep changes focused on one behavior area.
2. Add or update validation coverage for the public contract.
3. Update `README.md` when the public API changes.
4. Record release-facing changes in `CHANGELOG.md`.
